# 問題点と改善提案

## 実施済みの改善
- モジュール分離: math / object / material / scene / renderer / main-config の責務を分離。
- インタラクティブ操作: SDLイベントでカメラ移動し、入力時のみ再レンダリングする構成を実装。
- 性能改善: renderer.cpp で OpenMP schedule(dynamic) による並列描画を実装。
- 表現拡張: 透過（屈折）・金属・拡散・発光と、太陽光による直接光計算を導入。

## 残課題（次の改善候補）

1) 交差判定の高速化
  - BVH などの空間分割を導入し、オブジェクト数増加時の探索コストを削減する。

2) シーン表現の拡張
  - 球体以外のプリミティブ（平面・三角形メッシュ）を Object 実装として追加する。

3) 設定外部化
  - 現在のヘッダ直書き設定を JSON/TOML などの外部ファイル経由で読み込めるようにする。

4) 品質保証の強化
  - 交差判定、散乱計算、色変換の単体テストと、レンダリング時間のベンチマークを整備する。
